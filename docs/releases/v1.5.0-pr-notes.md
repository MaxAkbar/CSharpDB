## Highlights

- Added `Database.PrepareInsertBatch()` and `InsertBatch` for reusable low-allocation batch inserts.
- Added SQL fast paths for simple `INSERT INTO ... VALUES (...)` statements, including multi-row values.
- Added prepared ADO.NET insert binding for compatible `CSharpDbCommand` executions.
- Reduced lookup and page-maintenance allocations in the storage engine through zero-copy payload reads and span-based defragmentation/rebalance copies.
- Updated the macro batch benchmark harness to exercise the engine batcher.

## Benchmark Deltas

- SQL single insert: `21.1K ops/sec` published -> `22.17K ops/sec` current median-of-3
- SQL batched insert: `~294K rows/sec` published -> `~605K rows/sec` current median-of-3
- SQL point lookup: `564K-1.361M ops/sec` published -> `1.151M ops/sec` current median-of-3
- SQL concurrent reads (8 readers): `219K ops/sec` published -> `250K ops/sec` current median-of-3
- Collection point lookup: `1.227M ops/sec` published -> `1.320M ops/sec` current median-of-3
