## Notes for Reviewers

- `ICSharpDbClient` now exposes `GetMaintenanceReportAsync()`, `ReindexAsync(...)`, and `VacuumAsync()`. Those operations flow through direct and gRPC transports, the CLI, the Web API maintenance endpoints, and the Admin storage tab.
- `CSharpDB.Daemon` is now the dedicated gRPC host. `CSharpDB.Client` has explicit gRPC transport support for SQL, schema browsing, collections, procedures, maintenance, and storage-inspection flows.
- Collection indexed reads avoid unnecessary document materialization, and indexed row-id maintenance is shared across collection and SQL paths through `RowIdPayloadCodec`.
- Storage tuning now has opt-in presets via `UseLookupOptimizedPreset()` and `UseWriteOptimizedPreset()`. The write preset keeps auto-checkpoint work off the triggering commit with background sliced execution, but it does not change the default checkpoint policy.
- Benchmark coverage was expanded with `CollectionIndexBenchmarks`, `StorageTuningBenchmarks`, and `DurableWriteDiagnosticsBenchmark`, and the benchmark README was refreshed to the latest reproducible macro and write-diagnostics snapshots.
