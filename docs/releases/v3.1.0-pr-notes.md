## Summary

This `v3.1.0` release ships three major themes: continued multi-writer and
durable-write engine work after the initial phase-1 base, transport/runtime
integration updates around ADO.NET and the daemon host, and user-facing
docs/sample additions including Atlas Platform Showcase, the CSV bulk import
tutorial/sample, and the multi-writer follow-up plan.

On the engine side, this branch extends the original phase-1 multi-writer work
with stronger explicit `WriteTransaction` handling, richer logical conflict
tracking, disjunctive and range predicate validation, more aggressive insert-side
rebase and split recovery, phase-3 snapshot checkpoint retention tuning, and
phase-4 shared auto-commit fan-in for more write shapes. The branch also adds
the opt-in `ImplicitInsertExecutionMode = ConcurrentWriteTransactions` path,
insert fan-in diagnostics, commit-path diagnostics, and a large amount of test
and benchmark coverage around concurrent durable writes and hot insert behavior.

The same release line also contains a substantial performance and plumbing pass.
Insert commit allocation hot paths, single-insert commit cost, collection
secondary-index maintenance, and reader-session `COUNT(*)` behavior all received
focused optimization work. WAL publication and flush staging were split and then
instrumented more explicitly, group-commit tuning/docs were refreshed, and the
benchmark suite plus checked-in baselines were expanded around checkpoint
retention, insert fan-in, commit fan-in, and related concurrency behavior.

Outside the core engine, the branch routes the ADO.NET provider through
`CSharpDB.Client`, adds the supporting remote session infrastructure, and moves
the daemon host defaults toward hybrid multi-writer mode. It also adds the Atlas
Platform Showcase sample, the new API-first CSV bulk import sample and tutorial,
and a forward-looking multi-writer follow-up planning document linked from the
roadmap. README, architecture, configuration, FAQ, internals, ecosystem,
benchmark, and website docs were refreshed to match the current shipped state.

## Notes for Reviewers

- Review this as a broad multi-writer/durable-write branch, not as a small docs
  release. The CSV bulk import sample and the multi-writer follow-up plan are
  just the latest user-facing additions on top of a much larger runtime branch.
- The highest-risk code remains in `Pager`, `WriteAheadLog`,
  `TransactionCoordinator`, `Database`, `WriteTransaction`, and the insert-side
  rebase/split helpers. The key checks are durable-write correctness,
  fan-in behavior, retry stability, retained-WAL/checkpoint behavior, and
  insert-path regression risk under concurrency.
- The transport-side review should focus on the ADO.NET-to-client routing and
  the daemon’s newer host-database defaults.
- The user-facing docs/sample work is intentionally educational and should be
  reviewed as documentation of the current supported bulk-ingest path, not as a
  new first-class `bcp`/`COPY` product feature.
