## Notes for Reviewers

- The VS Code extension is a real product feature in this release, not just repo scaffolding: it runs locally in the VS Code extension host and talks directly to `CSharpDB.Native` through the embedded C API.
- The extension scope in this release includes workspace auto-connect, schema explorer, `.csql` language support, query results, data browser CRUD, table designer flows, and storage diagnostics.
- The in-memory storage fix is intentionally narrow: it changes growth behavior only when the next doubling step would exceed `Array.MaxLength`, and otherwise keeps the existing doubling strategy.
- The checkpoint diagnostics change is paired with that fix so any future in-memory WAL failure still carries page-count and byte-length context.
- The trim/AOT changes are primarily about surfacing the limitation at the correct public boundary and removing internal linker warnings, not about making `Collection<T>` fully NativeAOT-safe.
- The VS Code extension change is a config migration from deprecated `moduleResolution=node10` behavior to `Node16`.
- The SDK bump is kept as a separate release-prep concern so it can be evaluated independently of the runtime fixes if needed.
