## Summary

This `v2.9.0` release is focused on the interactive CLI experience in `CSharpDB.Cli`.

The branch replaces the older handwritten ANSI formatting layer with `Spectre.Console`, adds a shared console helper for prompts, status messages, panels, tables, and the startup banner, and moves the main dot-command, inspection, and maintenance surfaces onto richer structured console output. Query results now render through Spectre-backed tables, schema and object definitions use panels and formatted tables, and the CLI startup flow presents the new branded ASCII logo.

This release also adds the interactive dot-command menu requested for the REPL. At a fresh `csdb>` prompt, pressing `.` as the first character now opens a keyboard-driven list of supported dot commands with `Up`/`Down` selection, `Enter` to run, and `Esc` to cancel. The implementation keeps redirected and scripted input safe by falling back to normal non-interactive command handling when the shell is not attached to an interactive console.

Follow-up hardening in the same branch fixes the console-buffer crash uncovered during live testing. The menu renderer now clamps its draw region within the current buffer, limits the visible window on shorter terminals, and scrolls the visible portion of the command list instead of attempting to position the cursor outside the console bounds.

## Notes for Reviewers

- Review the CLI change as a user-experience and terminal-behavior release rather than an engine change. The key checks are prompt flow, output consistency, and whether the new menu logic behaves correctly across interactive and redirected sessions.
- The highest-risk area was the interactive `.` menu because it touches raw console key handling and cursor positioning. The important regression checks are near-bottom prompts, narrow terminals, and non-interactive input redirection.
- The `Spectre.Console` migration intentionally leaves command semantics alone. Most changes in `MetaCommands`, `InspectorCommandRunner`, `MaintenanceCommandRunner`, and `TableFormatter` are presentation-layer rewrites rather than behavioral changes to database operations.
