## Summary

This `v2.9.0` release is broader than the CLI changes alone. The branch carries four main tracks: phase-2 planner statistics and costing work, durable-write and copy-path batching, merge-follow-up stabilization for planner/checkpoint behavior, and a richer `CSharpDB.Cli` terminal experience built on `Spectre.Console`. It also includes a targeted performance recovery for cache-hot covered composite-index lookups after focused benchmark validation surfaced a regression against the current baseline.

On the engine side, this release deepens the optimizer’s statistics-guided behavior. `ANALYZE` now feeds richer internal planner data through histograms, heavy hitters, and composite-index prefix summaries, allowing better selectivity and join-cardinality estimates. The planner also expands join ordering with bounded dynamic programming across small reorderable inner-join chains instead of depending only on simpler greedy selection. This release line also makes row-count exactness explicit with `sys.table_stats.row_count_is_exact`, separating planner-advisory estimates from exact-count semantics so optimizer costing and `COUNT(*)` fast paths can behave correctly without weakening commit durability.

On the storage/runtime side, the branch adds shared batching infrastructure for durable-write and copy-style paths, including reusable storage copy batching and shared B-tree copy utilities used by maintenance and export-style operations. The intent is to reduce repeated per-page/per-row overhead in write-adjacent paths while keeping the default durability model intact. Follow-up commits in the branch then stabilize these changes around planner behavior, checkpoint interactions, and test regressions discovered after merging from `main`. A later perf-focused follow-up also restores covered composite-index lookup performance by exposing synchronous cache-only reads from the default B-tree index store and letting exact covered composite projections short-circuit directly from cached index payloads.

The release also refreshes the CLI presentation layer. `CSharpDB.Cli` now uses `Spectre.Console` instead of the earlier handwritten ANSI helper, with richer query result tables, help output, schema/definition panels, inspector and maintenance summaries, and a new branded ASCII startup banner. At the REPL, typing `.` as the first character at a fresh `csdb>` prompt now opens an interactive dot-command menu with `Up`/`Down` navigation, `Enter` to execute, and `Esc` to cancel. The menu path includes redirected-input fallback behavior and console-buffer clamping so it remains safe in scripted sessions and in short or nearly full terminals.

This release round also adds and refreshes contributor-facing documentation, including new configuration guidance, a query execution pipeline walkthrough, an expanded SQL reference, and focused notes on query and durable-write performance behavior.

## Notes for Reviewers

- Review this as a mixed optimizer/runtime/CLI release, not a single-area change. The branch is carrying substantial planner-statistics and durable-write work in addition to the terminal UX updates.
- The highest-risk engine areas are `QueryPlanner`, `CardinalityEstimator`, `CatalogService`, and the write-adjacent batching/copy paths. The important checks are that the richer stats improve planning without destabilizing existing correctness behavior, and that exact-versus-estimated row-count semantics remain clear.
- The covered composite-index fast path in `QueryPlanner` and `BTreeIndexStore` is intentionally narrow. The key review points are that cache-hot exact covered projections bypass operator setup without changing residual-predicate handling, `LIMIT`/`OFFSET` semantics, or non-covered lookup behavior.
- The follow-up stabilization commit is important context for this release. Regression checks around planner decisions, checkpoint behavior, and integration coverage should be read as part of the shipped scope rather than incidental cleanup.
- The CLI migration intentionally changes presentation more than behavior. The main behavioral addition on that side is the interactive `.` menu and its console-safety handling for interactive versus redirected sessions.
