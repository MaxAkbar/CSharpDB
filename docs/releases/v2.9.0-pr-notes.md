# Summary

The `v2.9.0` release introduces an admin-only forms stack inside `CSharpDB.Admin`.

This branch replaces the standalone form-designer prototype with a first-class `CSharpDB.Admin.Forms` module that lives inside the main solution and is hosted directly by the admin shell. The module includes the visual designer, runtime form-entry surface, schema adaptation, validation inference, formula evaluation, JSON serialization, and active-database form persistence through the internal `__forms` metadata table.

The admin host now exposes Forms in navigation and tab management, including designer and entry tabs plus table-level affordances to open the designer from the existing shell. On the runtime side, the branch adds practical usability fixes discovered during integration work: record paging instead of loading entire tables into the right pane, direct go-to-record by primary key, column-based search, manual child-table mapping when no declared foreign key exists, and control-value coercion so checkbox/radio bindings work against text- and integer-backed boolean fields.

The release also ports and expands the dedicated test coverage into `CSharpDB.Admin.Forms.Tests`, covering repository persistence, schema mapping, record services, serialization round-trips, generator/validation behavior, relation resolution, and admin tab integration.

## Notes for Reviewers

- Review this as an integration-heavy admin feature, not just a new Razor library. The important checks are admin-shell wiring, `__forms` persistence behavior, schema-signature handling, and runtime record-flow usability on nontrivial tables.
- The shipped scope is intentionally admin-only. This removes the standalone-host assumption from the prototype and keeps persistence local to the active database instead of adding public API surface.
- Child relationships now support both declared foreign keys and explicit manual mapping, but the runtime scope remains single-column parent/child mapping in this release.
- Full-solution release validation has not been rerun as part of this snapshot; the dedicated forms build and test suite were executed instead.
