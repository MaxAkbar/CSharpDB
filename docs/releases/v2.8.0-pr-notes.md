# Summary

The `v2.8.0` release is centered on one main runtime track: a batch-first SQL row transport foundation across the execution engine.

This branch pushes batch-native execution deeper into the existing operator pipeline so batch-capable scans, joins, projections, filters, limits, and generic aggregate paths can keep work in row batches instead of repeatedly materializing single rows. The result is primarily an internal architecture release rather than a syntax-heavy one, but it meaningfully lowers per-row overhead, broadens reusable predicate/projection kernels, and creates the base needed for later vectorized and more specialized execution work.

Release prep also hardened this runtime path around compact-row evaluation and numeric-expression predicate binding, which fixed batch-plan regressions discovered during the validation sweep. Outside the engine, the branch adds new design docs for compiled live queries and materialized join read models, ships new site blog/news pages plus roadmap refreshes, and includes a small Admin UI plumbing refactor around the title bar, modal flow, and shared database client state.
