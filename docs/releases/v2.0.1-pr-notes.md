# Summary

The `v2.0.1` pass delivers the Visual Query Designer for `CSharpDB.Admin` — a classic source canvas, SVG join lines, design grid, live SQL preview, and saved layouts, all embedded as a mode toggle inside the existing query tab.

The designer is fully contained within `CSharpDB.Admin` with no changes to the engine, client, API, or gRPC surface.

## Related Issues

- Visual Query Designer for `CSharpDB.Admin` — source canvas with draggable table nodes, SVG bezier join lines, design grid, and live SQL preview inside the existing query tab.

## Notes for Reviewers

- The designer lives entirely in `CSharpDB.Admin`. No engine, client, API, or gRPC changes are included in this pass.
- Designer mode is a toggle on the existing `QueryTab`. The `QueryMode` enum (`Sql`, `Designer`) and the current mode are stored in `TabDescriptor.State["QueryMode"]` so the tab restores correctly when re-activated.
- `QueryDesignerPanel` is a self-contained `IDisposable` Razor component. It receives a `TabDescriptor` and an `EventCallback<string> OnCopySql`. All designer state is local to the component and persisted via `TabDescriptor.DesignerStateJson` on every state change.
- `QueryDesignerSqlBuilder.Build` is a pure static method. It generates `SELECT *` when no columns are checked, qualifies all column references with the table name, combines filter expressions with `AND`, and respects `SortOrder` for multi-column `ORDER BY`.
- Saved designer layouts use the `__designer_layout:` name prefix in `__saved_queries`. `QueryTab` filters these out of the SQL-mode saved-query dropdown using a `StartsWith` check.
- JS drag uses document-level listeners set up once in `OnAfterRenderAsync(firstRender: true)`. Subsequent renders do not re-attach or tear down listeners. The `.NET` `OnTableMoved` callback fires on `mouseup` and updates the server-side node position; by that point the node's `style.left` / `style.top` already match, so Blazor's DOM diffing produces no visible jump.
- The backdrop (`position: fixed; inset: 0; z-index: 150`) appears only when the Add Table dropdown, Load dropdown, or join edit popup is open. It intercepts the next click to dismiss those panels without any canvas `@onclick` handler that could race against `OnTableMoved`.
- The `designerInterop._initialized` flag prevents double-initialization if the component is remounted (e.g. tab switch). The `_dotNetRef` is updated on every `initDrag` call so the document-level listeners always call back to the current live component instance.
- CSS uses only existing custom properties (`--bg-secondary`, `--border-color`, `--accent-blue`, etc.) — no hardcoded colors.
- v1 scope: single `SELECT` with projection, filtering, and sorting. `GROUP BY`, `DISTINCT`, `LIMIT`, subqueries, and SQL-to-designer import are explicitly deferred to a later pass.
