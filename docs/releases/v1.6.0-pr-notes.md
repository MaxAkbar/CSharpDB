## Highlights

- Added first-class engine in-memory open/load/save APIs and memory-backed storage/WAL implementations.
- Added ADO.NET `:memory:` and named shared-memory connection targets with explicit save/load support.
- Added collection-native secondary indexes plus direct UTF-8 collection payload encoding.
- Reduced collection write overhead by narrowing root persistence and skipping no-op root updates for non-indexed steady-state writes.
- Reused snapshot pager and planner state inside `ReaderSession` to improve concurrent read throughput for burst workloads.
- Expanded benchmarks to cover in-memory runtime, persistence, cold lookups, collection path costs, and reader scaling.
- Added a storage tutorial track and runnable study examples project under `docs/tutorials/storage`.

## Benchmark Deltas

- SQL in-memory single insert: **~307K ops/sec**
- SQL in-memory batched insert: **~1.65M rows/sec**
- Collection in-memory single put: **~494K ops/sec**
- Collection in-memory batched put: **~1.01M docs/sec**
- SQL cold lookup: **~31K ops/sec file-backed** -> **~400K ops/sec in-memory**
- Collection cold lookup: **~29K ops/sec file-backed** -> **~375K ops/sec in-memory**
- Collection single put: **27.49K ops/sec** previous snapshot -> **29.74K ops/sec**
- SQL concurrent reads (8 readers): **~236K ops/sec** with per-query sessions, **~3.69M ops/sec** with reused reader sessions (`x32` reads per snapshot)
