Operations

Inspect, Maintain, Import, and Export

Studio includes operational surfaces for storage diagnostics, table archive import/export, external tables, pipelines, procedures, and host callback readiness.

Storage Inspector

The Storage tab shows low-level database health and maintenance information. Use it when diagnosing file size, WAL behavior, page distribution, index health, or maintenance actions.

Storage Inspector shows database header values, WAL state, page usage, fragmentation, integrity findings, and maintenance controls.

PanelUse it for
Database HeaderCheck page size, page count, schema root, freelist state, version, and file metadata.
WALReview write-ahead log existence, frame count, commit frames, and integrity status.
Space UsageCompare declared pages, physical pages, WAL size, free pages, and fragmentation indicators.
MaintenanceRun scoped reindex, vacuum, backup, restore, and foreign-key retrofit checks where available.
Page Drill-DownInspect a page by id, optionally including raw hex output.

Import and Export

Use Import / Export for table archive workflows, backups of selected data, restore operations, and external table registration. Long-running actions show progress and are designed to avoid blocking the rest of the workspace.

Pipelines

The Pipeline tab is a visual ETL builder for repeatable source-to-destination workflows.

The pipeline designer lets you configure source, transforms, destination, validation, dry-run, and execution from one tab.

  • Choose a source such as CSV, JSON, table, or SQL query.
  • Add transforms such as select, rename, cast, filter, derive, and deduplicate.
  • Validate before execution to catch shape errors.
  • Use dry run before writing destination data.

Procedures and Code Modules

Procedure tabs let you define metadata, SQL body, parameters, and test arguments. Code module and callback surfaces help coordinate trusted host-owned C# functions, commands, validation rules, and form/report automation hooks.

The procedure editor combines definition, parameters, and test execution in one view.

Operational Safety

  • Destructive commands ask for confirmation.
  • Metadata readiness panels surface missing callbacks before runtime use.
  • Preview and dry-run commands are available for pipelines and reports where supported.
  • Use backups before large schema, import, restore, or maintenance operations.
