Forms and reports

Build Operational Screens and Printable Output

Studio includes Access-style designers for data entry forms and banded reports. Both store metadata in the database and can be opened from Object Explorer.

Form Designer

Create forms from the title bar, from a table context menu, or from the Forms group in Object Explorer. The designer uses toolbox, canvas, layers, and property inspector panels.

The form designer supports drag and drop layout, breakpoint-specific positioning, property editing, validation, and action sequences.

AreaPurpose
ToolboxAdd labels, text inputs, numbers, dates, checkboxes, selects, lookups, computed fields, child grids, and child tabs.
CanvasPosition controls visually, resize them, use snapping, and preview layout.
LayersSelect, reorder, and inspect controls in the form hierarchy.
PropertiesConfigure binding, appearance, validation overrides, control events, and action sequences.

Form Data Entry

Open the runtime data entry view from the designer toolbar or from Object Explorer. Runtime forms provide record navigation, search, save/delete, validation, printing, and a jump back to design mode.

Runtime forms combine a rendered form, record list, navigation, search, and validation feedback.

Reports

Reports are printable, banded layouts bound to tables, views, or saved queries. Use them for operational summaries, invoices, labels, and review packets.

Report bands

Use report header/footer, page header/footer, group header/footer, and detail bands to control where content renders.

Controls

Place labels, bound text, calculated text, lines, and boxes on the canvas.

Expressions

Calculated text supports field references, arithmetic, page number, print date, and aggregate functions such as SUM and COUNT.

Preview and print

Render reports with live data, navigate pages, refresh output, and use browser printing.

Automation and Callbacks

Form and report metadata can reference host-owned callbacks for validation, command execution, and lifecycle events. Missing callbacks are visible in readiness and diagnostics surfaces so app builders and host developers can coordinate safely.

Host-owned code: database metadata stores names and action data. The trusted C# implementation belongs in the host project startup registration.
