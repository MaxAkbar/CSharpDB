Workspace

Navigate the Studio Workspace

Studio uses a dense desktop layout: a title bar for global commands, Object Explorer for database objects, a tab strip for active work, and a status bar for connection context.

Title Bar

The title bar contains the current database, the command palette, common creation commands, theme toggle, sidebar toggle, and keyboard shortcut help.

AreaUse it for
Database switcherOpen or switch to another database path.
Search anythingOpen the command palette for tables, forms, reports, procedures, callbacks, and common commands.
New Query / Table / Form / ProcedureCreate common objects without navigating through Object Explorer.
Theme and layout buttonsToggle dark/light mode, hide the sidebar, or show shortcuts.

Object Explorer

Object Explorer is the left navigation tree. It groups user tables, external tables, collections, forms, reports, views, procedures, callbacks, storage tools, and pinned or recent items.

  • Use filter chips to narrow the tree by object type.
  • Use the search box to filter object names.
  • Click an object to open its default tab.
  • Right-click objects for contextual actions such as design, preview, copy name, import/export, or delete.
  • Use Refresh when external tools or another process may have changed metadata.

Tabs

Each task opens in a tab. Tabs preserve local state, so a query, form designer, report preview, and storage scan can remain open at the same time.

  • Click a tab to activate it.
  • Use the close button to close a tab.
  • Press Ctrl+W to close the active closable tab.
  • Use recent items in Object Explorer to reopen frequently used tabs.

Command Palette

Open the command palette with Ctrl+K. It is the fastest way to jump to known objects, run common commands, create objects, or return to recent work.

A query tab combines a SQL editor, toolbar actions, saved queries, execution status, and results.
