ETL Pipelines

Built-in pipeline runtime for Extract-Transform-Load workflows. Define pipelines as JSON packages, execute with validation and dry-run, and track run history.

Overview

The pipeline system connects sources to destinations through a chain of transforms. Pipelines are defined as JSON package files and executed by the PipelineOrchestrator.

Sources

SourceDescription
CsvPipelineSourceRead rows from CSV files with configurable delimiters and headers
JsonPipelineSourceRead objects from JSON files or arrays
SQL QueryExecute a SELECT statement and stream results
CSharpDB TableRead directly from a database table

Transforms

TransformDescription
ProjectionSelect specific columns to pass through
Column RenameRename columns in the data flow
Type CoercionConvert column types (e.g., text to integer)
Row FilterInclude/exclude rows based on conditions
Derived ColumnCompute new columns from expressions
DeduplicationRemove duplicate rows by key columns
OrderedSort rows by specified columns

Destinations

DestinationDescription

Pipeline Package Definition

Pipelines are defined as JSON packages that describe the source, transforms, and destination.

pipeline.json 
{
  "name": "import-products",
  "source": {
    "type": "csv",
    "path": "products.csv",
    "hasHeader": true
  },
  "transforms": [
    { "type": "rename", "mappings": { "product_id": "Id", "product_name": "Name" } },
    { "type": "coerce", "columns": { "Id": "integer", "Price": "real" } },
    { "type": "filter", "expression": "Price > 0" }
  ],
  "destination": {
    "type": "table",
    "table": "Products"
  }
}

Execution Modes

ModeDescription
ValidateCheck the pipeline definition for errors without executing
Dry RunSimulate execution, reporting row counts and potential issues
ExecuteRun the full pipeline, writing to the destination
ResumeResume a failed pipeline from the last checkpoint

Management

Pipelines can be managed through multiple interfaces:

  • Admin UI — Visual pipeline designer with drag-and-drop nodes, connection lines, and properties panel
  • REST API — Pipeline CRUD and execution endpoints
  • Client SDK — CSharpDbPipelineRunner and CSharpDbPipelineStorage classes
  • CLI — Command-line pipeline execution and inspection

Run History

Every pipeline execution is tracked with metadata including start/end times, row counts, error details, and reject records. Failed rows are captured for inspection and recovery.
